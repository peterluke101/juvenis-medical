# New Client Onboarding Checklist
# Copy this folder to clients/[client-slug]/ for every new client

---

## Step 1 — Intake (Day 1)
- [ ] Get client's legal business name
- [ ] Get primary contact name, email, phone
- [ ] Get website URL (or confirm we're building one)
- [ ] Confirm direct intake form / CRM URL for all CTAs
- [ ] Confirm social accounts: Instagram, Facebook, TikTok, LinkedIn, Google Business
- [ ] Confirm service area and target audience (age, gender, location)
- [ ] Confirm any compliance requirements (HIPAA, FDA, etc.)
- [ ] Get brand assets: logo, colors, fonts (if they have them)

## Step 2 — Setup (Days 1-3)
- [ ] Create client folder: `clients/[slug]/`
- [ ] Add to `clients/roster.md`
- [ ] Add to `clients/dashboard.md`
- [ ] Set up Stripe subscription
- [ ] Send social media access request email with instructions
- [ ] Add client to Buffer once access granted
- [ ] Create `social/` folder structure from template
- [ ] Update `social/social-strategy.md` with client details

## Step 3 — Content Kickoff (Week 1)
- [ ] Assign first two blog topics (1 per audience)
- [ ] Generate all 30 social posts
- [ ] Run /medical-wording if medical client
- [ ] Send to client for approval
- [ ] Run Content Asset Scout for image library
- [ ] Schedule first month of posts in Buffer

## Step 4 — Ongoing (Monthly)
- [ ] Pete assigns new topics by 1st of month
- [ ] Content generated and reviewed by 7th
- [ ] Client approves by 10th
- [ ] All posts scheduled by 12th
- [ ] Stripe auto-bills on anniversary date
- [ ] Monthly check-in with client (15 min)

---

## Files to Create for Each Client
- `project-notes.md` — goal, audience, services, CTA URL, credentials
- `session-notes.md` — running log of work done
- `social/social-strategy.md` — platform priorities, copy rules
- `social/queue.md` — content status tracker
- `social/2026-[MM]/calendar.md` — monthly content calendar
